# VFI-gui i18n 实现指南

## 概述

本文档记录 VFI-gui 国际化系统的实现细节和最佳实践。

## 技术选型

- **gettext** - Python 标准库，行业标准
- **polib** - PO/MO 文件处理库
- **自定义 MO 加载器** - 解决 gettext.GNUTranslations 的 UTF-8 编码问题

## 实现细节

### 1. 初始化顺序

**关键：** i18n 必须在 UI 导入之前初始化。

```python
# main.py
from core.i18n import init_i18n
init_i18n(str(APP_DIR / "locales"))  # 先初始化

from ui.main_window import MainWindow  # 再导入 UI
```

### 2. 自定义 MO 加载器

Python 标准库的 `gettext.GNUTranslations` 在处理 UTF-8 编码的 MO 文件时会报错：

```
UnicodeDecodeError: 'ascii' codec can't decode byte 0xe7
```

解决方案：自定义 MO 文件解析器，直接使用 UTF-8 解码。

```python
def _load_mo_file(self, mo_path: Path):
    """自定义 MO 文件加载，正确处理 UTF-8"""
    import struct
    
    with open(mo_path, "rb") as f:
        mo_data = f.read()
    
    # 解析 MO 文件结构
    magic = struct.unpack_from("<I", mo_data, 0)[0]
    num_strings = struct.unpack_from("<I", mo_data, 8)[0]

```

### 3. UI 组件重翻译

所有 UI 组件实现 `retranslate_ui()` 方法：

```python
class VideoInputWidget(QWidget):
    def _setup_ui(self):
        # 保存需要重翻译的控件引用
        self.group = QGroupBox(tr("Video Input"))
        self.browse_btn = QPushButton(tr("Browse..."))
    
    def retranslate_ui(self):
        """语言切换时更新 UI 文本"""
        self.group.setTitle(tr("Video Input"))
        self.browse_btn.setText(tr("Browse..."))
```

主窗口连接语言切换信号：

```python
def _setup_connections(self):
    get_i18n().language_changed.connect(self._on_language_changed_signal)

def _on_language_changed_signal(self, lang):
    self._retranslate_ui()

def _retranslate_ui(self):
    # 更新窗口标题
    self.setWindowTitle(tr("VFI-gui - Video Frame Interpolation"))
    # 更新菜单栏
    self.menuBar().clear()
    self._setup_menubar()
    # 更新所有子组件
    self.video_input.retranslate_ui()
    self.pipeline_config.retranslate_ui()
    self.progress_panel.retranslate_ui()
    self.batch_queue.retranslate_ui()
```

### 4. 语言偏好持久化

```python
# config.py
DEFAULT_SETTINGS = {
    "ui": {
        "language": "",  # 空表示自动检测系统语言
    },
}

def get_language(self) -> str:
    return self.settings.get("ui", {}).get("language", "")

def set_language(self, language: str):
    if "ui" not in self.settings:
        self.settings["ui"] = {}
    self.settings["ui"]["language"] = language
```

## 翻译工作流

### 1. 添加新翻译字符串

编辑 `locales/messages.pot`：

```po
msgid "New String"
msgstr ""
```

### 2. 更新各语言翻译

编辑 `locales/zh_CN/LC_MESSAGES/messages.po`：

```po
msgid "New String"
msgstr "新字符串"
```

### 3. 编译 MO 文件

```bash
python compile_translations.py
```

输出：
```
Compiling: zh_CN\LC_MESSAGES\messages.po
  -> zh_CN\LC_MESSAGES\messages.mo
Compiled 2 translation files
```

### 4. 在代码中使用

```python
from core import tr
label = QLabel(tr('New String'))
```

## 最佳实践

### DO

- ✅ 在 UI 组件中保存需要翻译的控件引用
- ✅ 实现 `retranslate_ui()` 方法
- ✅ 使用 `tr()` 包裹所有用户可见文本
- ✅ 编译 MO 文件后再运行应用

### DON'T

- ❌ 在模块级别调用 `tr()` (此时 i18n 未初始化)
- ❌ 硬编码用户界面文本
- ❌ 忘记编译 MO 文件
- ❌ 使用 `gettext.GNUTranslations` 直接加载 MO 文件

## 常见问题

### Q: 翻译不生效？

检查：
1. MO 文件是否存在且最新
2. `init_i18n()` 是否在 UI 导入前调用
3. PO 文件中翻译是否非空

### Q: 语言切换后部分文本未更新？

检查：
1. 该组件是否实现了 `retranslate_ui()`
2. 主窗口是否调用了该组件的 `retranslate_ui()`
3. 控件引用是否正确保存

### Q: MO 文件加载报编码错误？

使用自定义 MO 加载器 (`_load_mo_file`)，而非 `gettext.GNUTranslations`。
